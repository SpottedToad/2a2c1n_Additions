  SpaceAxolotlTracker.java (Interface): Concrete public interface applied to AxolotlEntity via Mixin. Hosts public getSpaceVariant() and setSpaceVariant(int) getters and setters to handle data states safely without type inference or package-private crashes.

  AxolotlEntityMixin.java (Server Data & Logic):
    Initializes a @Unique integer DataTracker tracking our custom variant index (0 = Vanilla, 1 = Space Variant).
    Injects NBT serialization hooks inside writeCustomDataToNbt and readCustomDataFromNbt for flawless data persistence across chunk unloads and world reloads.
    Intercepts copyDataToStack and copyDataFromNbt to preserve variant states when entities are collected or deployed via standard water buckets.
    AxolotlEntityRendererMixin.java (Client Assets): Injecting at @At("HEAD") inside getTexture allows our system to completely intercept vanilla's skin lookup chain. It dynamically routes the engine to adult (space_variant.png) or baby (space_variant_baby.png) asset textures based on active data states and isBaby() queries.
    acn_additions.accesswidener (Access Rules): Configured using tab-separated layout mapping targeting Yarn bytecode configurations. This opens AxolotlEntity.setVariant and its VARIANT data tracker fields to public visibility, resolving internal private class restrictions. Linked directly via build.gradle inside the Loom configuration scope block.

        Purebred Breeding: Two space variants guarantee a 100% space variant offspring.
